# PRODUCT

Last updated: 2026-05-15

## 1. Product summary

NeveraChef-AI helps users and families cook with what they already have by transforming ingredient input into actionable recipes and shopping outputs.

## 2. Problem statement

Users commonly face:

- Unclear meal decisions with available ingredients.
- Food waste from unused products.
- Repetitive cooking routines.
- Poor shopping planning.

## 3. Target users

- Families (including households with children).
- Users seeking healthier meals.
- Users seeking lower-cost meal planning.
- Users wanting fast, realistic recipes.

## 4. Confirmed MVP features

1. Manual ingredient input.
2. Ingredient review/edit step.
3. Basic user preferences.
4. AI-generated recipes from available ingredients.
5. Recipe detail with used vs missing ingredients.
6. Shopping list generation from missing ingredients.

## 5. Future features (not MVP-blocking)

- Voice input.
- Camera/image ingredient detection.
- Weekly menu advanced planning.
- History/favorites.
- Cloud sync and account system.

## 6. MVP boundaries

Out of initial MVP:

- Login/auth.
- Own backend platform.
- Payments/subscriptions.
- Full production release hardening scope.

## 7. Open product questions

1. Is weekly menu part of MVP acceptance or first post-MVP increment?
2. What is the minimum "successful AI output" threshold for Sprint 3 demo?
3. What is the data retention UX for local-only storage in settings?

## 8. Acceptance criteria for MVP success

MVP is considered successful if users can:

1. Add ingredients manually.
2. Adjust preferences.
3. Generate recipes using AI text flow.
4. View recipe details and missing ingredients.
5. Build and manage a shopping list.
